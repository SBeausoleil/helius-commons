# V0.5.0
Introduces the Result class and a refactoring of the introspection mechanism.

## Breaking changes
Since the project is not yet in 1.0.0 state, some breaking changes are allowed, though we do our best to avoid them.

- **The method BeanIntrospector.seek(Class<T> targetType, Object root, Lookup context) now throws an IntrospectionException not an IllegalAccessException.**
This is to reflect the wider variety of possible failure causes now that we aim to allow more freedom in the means of traversing objects graphs.
